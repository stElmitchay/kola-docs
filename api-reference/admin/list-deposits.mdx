## List Deposits (Admin)

Returns every deposit for the application identified by `X-Application-Id`, regardless of which API key created it. Use this for back-office views and reconciliation.

For partner-scoped access using an API key, see [`GET /v1/deposits`](/api-reference/deposits/list) instead.

### Headers

<ParamField header="X-Admin-Key" type="string" required>
  Your admin secret key.
</ParamField>

<ParamField header="X-Application-Id" type="string" required>
  The application whose deposits to list.
</ParamField>

### Query parameters

<ParamField query="status" type="string">
  Filter by status: `pending`, `confirmed`, `processing`, `completed`, `failed`, `expired`.
</ParamField>

<ParamField query="limit" type="integer">
  Maximum number of results (default: all).
</ParamField>

<ParamField query="offset" type="integer">
  Number of results to skip (for pagination).
</ParamField>

<RequestExample>
```bash All deposits
curl https://kola-api-beta.vercel.app/api/v1/admin/deposits \
  -H "X-Admin-Key: YOUR_ADMIN_KEY" \
  -H "X-Application-Id: a7784b0d-8be0-4338-be20-1d1a3ac9d1fc"
```

```bash Filtered
curl "https://kola-api-beta.vercel.app/api/v1/admin/deposits?status=completed&limit=10" \
  -H "X-Admin-Key: YOUR_ADMIN_KEY" \
  -H "X-Application-Id: a7784b0d-8be0-4338-be20-1d1a3ac9d1fc"
```
</RequestExample>

### Notes

- Results are ordered by `created_at` descending (newest first).
- Field names match the database snake_case schema (unlike the camelCase single-row responses from `GET /v1/deposits/:id`).
- `phoneNumber` is not a supported filter on this endpoint — use `status`, `limit`, and `offset` only.

### Errors

| Status | Message |
|--------|---------|
| 400 | `X-Application-Id header is required` |
| 401 | `Missing admin key` / `Invalid admin key` |
| 500 | Database error (message from upstream) |
