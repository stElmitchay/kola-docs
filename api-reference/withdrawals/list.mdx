## List Withdrawals

Returns withdrawals for your application, newest first. Supports filtering by phone number and status.

### Headers

<ParamField header="X-API-Key" type="string" required>
  Your API key.
</ParamField>

### Query parameters

<ParamField query="phoneNumber" type="string">
  Filter by phone number (normalized format, e.g., `+23276123456`).
</ParamField>

<ParamField query="status" type="string">
  Filter by status: `pending_deposit`, `received`, `processing`, `completed`, `failed`, `expired`.
</ParamField>

<ParamField query="limit" type="integer">
  Maximum number of results (default: all).
</ParamField>

<ParamField query="offset" type="integer">
  Number of results to skip (for pagination).
</ParamField>

<RequestExample>
```bash All withdrawals
curl https://kola-api-beta.vercel.app/api/v1/withdrawals \
  -H "X-API-Key: YOUR_API_KEY"
```

```bash Filtered
curl "https://kola-api-beta.vercel.app/api/v1/withdrawals?status=completed&limit=10" \
  -H "X-API-Key: YOUR_API_KEY"
```
</RequestExample>

### Notes

- Results are ordered by `created_at` descending (newest first).
- Each application only sees its own withdrawals.
- Field names match the database snake_case schema (unlike the camelCase single-row responses).
